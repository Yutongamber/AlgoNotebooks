DQN

1. 算法介绍

a. DQN研究背景、理论基础是什么？

Policy iteration；算法Q-learning

b. DQN研究动机是什么？

Q-learning 算法是以矩阵的方式建立了一张存储每个状态下所有动作值的表格。表格中的每一个动作价值估计的是在状态下选择动作然后继续遵循某一策略预期能够得到的期望回报。然而这种用表格存储动作价值只能适用于环境的状态和动作是离散的，并且空间都比较小的时候。但是，当状态或者动作数量非常大的时候，tabular 这种表示方式就不适用了。

例如，当状态是一个 RGB 图像时，假设图像大小是210×160×3，此时状态的数量一共有种256(210×60×3)，在计算机种存储这个数量级的 Q 表格是不现实的。更甚者，当状态或者动作是连续的时候，就有无限个状态动作对，我们更加无法使用这种表格的形式来记录各个状态动作对的 Q 值。面对这种情况，我们需要用函数拟合的方法来进行估计。DQN 算法的提出便来解决连续状态下离散动作的问题，也是深度强化学习的开山之作。

c. 提出的方法是什么？

1.DQN使用神经网络：通过参数化函数 (Function Approximation) 的方式来拟合动作价值函数 Q(s, a)。

作用：能够解决连续状态、状态空间大的问题

2.经验回放 (Experience Reply) : 在监督学习中，有一个非常重要的假设是训练数据是独立同分布的。假设训练数据是独立同分布的，我们每次训练神经网络的时候是从训练数据中随机采样一个或若干个数据来进行梯度下降，随着学习的不断进行，每一个训练数据会被使用多次。

训练神经网络需满足数据时独立同分布的假设，否则造成神经网络不稳定。在原先的 Q-learning 算法中，每一个数据只会用来更新一次 Q 值。于是，为了更好的将 Q-learning 和深度神经网络结合，DQN 算法采用了经验回放的策略。具体做法为维护一个回放池（Experience Buffer），从环境中采样到四元组数据（状态-动作-下一状态-奖励）会存储到回放池中，训练网络的时候再从回放池中随机采样若干数据来进行训练。

作用：一是在 MDP 中采样得到的数据本身不满足独立的假设，因为这一时刻状态和上一时刻的状态有关，采用经验回放可以打破样本之间的相关性，让其满足独立假设。二是每一个样本可以被使用多次，提高样本利用率。

名词解释：(Wiki)在概率论与统计学中，独立同分布 (Independent and identically distributed，或称独立同分配，缩写为iid、 i.i.d.、IID）是指一组随机变量中每个变量的概率分布都相同，且这些随机变量互相独立

3.增加一套网络 (Target Network)作为目标网络：用两个网络来处理时序差分中的偏差，一个网络（参数θ），另一个网络（参数θ′），分别作为当前动作价值函数Q(s,a)和目标动作价值函数，。目标网络与原训练网络结构相同但使用较旧的参数。然后每隔若干步，同步两个网络参数 θ′←θ。

作用：进一步打乱数据相关性。

"""
定义一个神经网络Qnet
网络设计：采用一层64个神经元的全连接，以ReLU作为激活函数
input:
    output_size: 动作空间大小
    input_size: 状态空间大小
    hidden_size: 隐藏层神经元个数，默认为64
    （num_hidden_layer: 增加隐藏层层数，默认为0）
"""


class QNet(nn.Module):
    def __init__(self, input_size, output_size, hidden_size=64, num_hidden_layer=0):
        super().__init__()
        self.input_size = input_size
        self.output_size = output_size
        self.num_hidden_layer = num_hidden_layer


        self.linear_default = nn.Sequential(
                        nn.Linear(input_size, hidden_size),
                        nn.ReLU())
        self.linear_out = nn.Linear(hidden_size, output_size)
        
        if self.num_hidden_layer > 0:
            hid_net = []
            for _ in range(self.num_hidden_layer):
                hid_net.append(nn.Linear(hidden_size, hidden_size))
                hid_net.append(nn.ReLU())
            self.linear_hid = nn.Sequential(*hid_net)


    def forward(self, x):
        x = self.linear_default(x)
        if self.num_hidden_layer > 0:
            x = self.linear_hid(x)
        x = self.linear_out(x)
        return x

"""
定义经验回放池
回放池数据更新原则：先进先出FIFO
数据采样：随机
"""


class Replay_buffer(object):
    def __init__(self, max_size):
        self.buffer_size = max_size
        self.data = list()
        
    def insert(self, state, action, reward, next_state, done):  
        if len(self.buffer) >= self.buffer_size:
            self.data.pop(0)
        self.data.append((state, action, reward, next_state, done))
    
    def sample(self, batch_size):
        transitions = random.sample(self.data, batch_size)
        state, action, reward, next_state, done = zip(*transitions)
        return np.array(state), np.array(action), np.array(reward), np.array(next_state), done
    
    def get_buffer_size(self):
        return len(self.data)

class DQN(object):
    def __init__(self, args):


        self.state_dim = args.obs_space
        self.action_dim = args.action_space


        self.hidden_size = args.hidden_size
        self.lr = args.c_lr
        self.buffer_size = args.buffer_capacity
        self.batch_size = args.batch_size
        self.gamma = args.gamma
        
        # 定义两个网络：
        self.q_eval = Critic(self.state_dim,  self.action_dim, self.hidden_size)
        self.q_target = Critic(self.state_dim, self.action_dim, self.hidden_size)
        self.optimizer = optimizer.Adam(self.critic_eval.parameters(), lr=self.lr)


        # exploration
        self.eps = args.epsilon
        self.eps_end = args.epsilon_end
        self.eps_delay = 1 / (args.max_episodes * 100)


        # 更新target网
        self.learn_step_counter = 0
        self.target_replace_iter = args.target_replace


        self.memory = Reply_buffer(self.buffer_size)


    # ========================== inference ==============================
    def choose_action(self, observation, train=True):
        if train:
            self.eps = max(self.eps_end, self.eps - self.eps_delay)
            if random.random() < self.eps:
                action = random.randrange(self.action_dim)


            else:
                observation = torch.tensor(observation, dtype=torch.float).view(1, -1)
                action = torch.argmax(self.critic_eval(observation)).item()
        else:
            observation = torch.tensor(observation, dtype=torch.float).view(1, -1)
            action = torch.argmax(self.critic_eval(observation)).item()


        return action


    # ========================== update ================================
    def learn(self):
        
        # 在回放池数据充满后，算法开始更新
        data_length = self.memory.get_buffer_size()
        if data_length < self.buffer_size:
            return
        state, action, reward, next_state, done = self.memory.sample(self.batch_size)
        
        obs = torch.tensor(state, dtype=torch.float)
        obs_ = torch.tensor(next_state, dtype=torch.float)
        action = torch.tensor(action, dtype=torch.long).view(self.batch_size, -1)
        reward = torch.tensor(reward, dtype=torch.float).squeeze()
        done = torch.tensor(done, dtype=torch.float).squeeze()


        q_eval = self.q_eval(obs).gather(1, action)
        q_next = self.q_target(obs_).detach()
        q_target = (reward + self.gamma * q_next.max(1)[0] * (1 - done)).view(self.batch_size, 1)
        loss_fn = nn.MSELoss()
        loss = loss_fn(q_eval, q_target)


        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()


        if self.learn_step_counter % self.target_replace_iter == 0:
            self.critic_target.load_state_dict(self.critic_eval.state_dict())
        self.learn_step_counter += 1


        return loss


    # ========================== save ================================
    def save(self, save_path, episode):
        model_q_path = "q_" + str(episode) + ".pth"
        torch.save(self.q_eval.state_dict(), model_critic_path)


    # ========================== load ================================
    def load(self, file):
        self.q_eval.load_state_dict(torch.load(file))

Gym Cartpole环境介绍

CartPole是一个杆子连在一个小车上，小车可以无摩擦的左右运动，杆子（倒立摆）一开始是竖直线向上的。小车通过左右运动使得杆子不倒。

马尔可夫决策过程 MDP

state:

状态空间：四维

  • 连续

sate_space = Box(4)

action

动作空间：二维

  • 离散

action_space = Discrete(2)

注意：速度的减少或增加的量不是固定的，因为它取决于极点指向的角度。这是因为杆子的重心增加了在它下面移动手推车所需的能量

reward:

rstep=1

如果保持平衡，每一步都给出1的奖励，包括终止状态。

done:
  • 杆子与竖直方向角度超过12度
  • 小车位置距离中心超过2.4（小车中心超出画面）
  • 片段长度超过200
  • 连续100次尝试的平均奖励大于等于195。
